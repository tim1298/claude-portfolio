Context

The opportunity, and the catch

Marketers want to target audiences based on nuanced behaviors and preferences. But analyzing individual attributes across millions of site visitors by hand is nearly impossible. Existing AI tools tried to help, and kept falling short:

  • Opaque — AI outputs were hard to explain
  • Unverifiable — no way to test or override a prediction
  • Risky — costly to apply at scale without confidence in accuracy
Problem

A "look what we can do" prototype, not a product

This was a feature-first project. Before I joined, the team had built a simulation tool that generated real-time attribute predictions, demographics, hobbies, brand preferences, from the tester's own browsing behavior. Impressive tech, no real-world utility.

  • Insights were based on biased simulations, not real user behavior
  • No practical use case beyond showcasing technical capability

The original build let a tester generate live attribute predictions from their own browsing, demographics, hobbies, brand preferences. Technically impressive, but built around the tester, not real customers, so the output couldn't be trusted or acted on.

Approach

Redefine the problem before designing the screens

Redefine the problem

I challenged the "simulation" assumption and led workshops to redirect the project toward generating attributes from real user behavior, not biased test data.

Create strategic value

That reframing turned a stagnant sandbox into a scalable marketing-intelligence tool, with a clear workflow from attribute definition to site-wide segmentation.

Align & mentor

I bridged PM, engineering, and UX around real user needs, advocated for human-centered design in technical discussions, and mentored junior designers throughout.

Building alignment & mentoring the team

I bridged PM, engineering, and UX around a shared vision centered on real user needs, advocated for human-centered design inside technical discussions so UX had a seat at the table, and mentored junior designers through information architecture, interaction patterns, and stakeholder communication.

How it works

A human-in-the-loop prediction workflow

01

Define the attribute

Marketers start by defining what they want to learn about their customers, for example, a user's preferred TV display technology.

02

Review sample predictions

They see a curated set of real user profiles with predicted values, confidence scores, and plain-language explanations, so they can test accuracy, give feedback, and refine the algorithm in a human-in-the-loop way.

03

View distribution & apply

Once the logic is refined, the system projects how the attribute distributes across the entire user base, giving a scalable view of segment potential. Confident in accuracy and reach, marketers apply it to launch targeted campaigns, like promoting OLED TVs to the users most likely to want them.

UX challenges

Designing for trust in AI

Challenge 01

How do we make the AI workflow intuitive?

I streamlined attribute creation, automated value generation, added CSV upload support, and combined the steps with real-time prediction examples so the workflow taught itself as users moved through it.

Challenge 02

How do we instill trust in AI predictions?

I built a sandbox live preview so marketers could see how predictions are generated in real time, and surfaced model confidence scores and explanations directly in the sample stage.

Challenge 03

How do we get users to give feedback that improves accuracy?

I reduced information overload by showing one user card at a time and prioritized clear feedback CTAs, making the step feel expected without feeling demanding.

Validated with clients

Assumptions we tested

  • What helps marketers trust AI predictions at first glance?
  • Which data points are most useful for verifying accuracy?
  • Are random or filtered demographic previews more insightful?
Results

From sandbox to scalable system

  • Established trust in AI tagging workflows
  • Reduced time-to-segmentation from days to minutes
  • Introduced a flexible system that's scalable and auditable
Takeaway

Building trust in AI takes more than "good" predictions, it takes transparency and control. Keeping a human in the loop inside an automated workflow didn't just protect accuracy; it made people willing to adopt the system at scale.
